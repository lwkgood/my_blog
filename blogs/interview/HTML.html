    # src || href 区别

    共同点

    • 引用外部资源

    不同点

    src：

    • 资源引用，资源下载并嵌入到标签位置，如请求js脚本（放页面底部）
    • 浏览器解析到它时，会暂停其他资源下载和处理，直到将该资源加载、编译、执行完成

    href：

    • 超文本引用，建立起网络资源和本文档的链接关系，如a、link标签
    • 浏览器解析到它时，会并行下载资源，不停止对当前文档的处理

    • 语义化：根据内容结构，选择合适标签

    # DOCTYPE 作用

    DOCTYPE

    • 概念：文档类型声明，声明在HTML文档第一行
    • 作用：告诉浏览器，应该以什么样的文档类型来解析文档，不同渲染模式会影响对css、js解析

    渲染页面模式

    • 标准模式：默认，浏览器使用W3C标准解析渲染页面，以其支持的最高标准呈现页面
    • document.compatMode 获取
    • 怪异模式：浏览器用自己的怪异模式解析渲染页面，页面以一种比较宽松的向后兼容的方式呈现

    # script标签中 defer 和 async 的区别

    script 没有 defer || async

    • 浏览器会立即加载并执行脚本，阻塞后续文档的加载

    defer || async

    • 都是异步加载外部JS脚本文件，都不会阻塞页面解析
    • defer：后续文档的加载和js脚本的加载（仅加载不执行）是并行进行的（异步）js脚本需要等到文档所有元素解析完成之后才执行（DOMContentLoaded事件触发执行之前）
    • async：后续文档的加载和执行与js脚本的加载和执行是并行进行的，异步执行
    多个带defer属性的标签，按照加载顺序执行

多个带async属性的标签，不能保证加载顺序

    # 说一下web worker

    原因、概念

    • HTML页面在执行脚本时，页面状态不可响应，直到脚本执行完成，才可响应
    • web worker 是运行在后台的js，独立于其他脚本，不影响页面性能，它通过postMessage将结果回传到主线程，这样不会阻塞

    如何创建？

    • 检查浏览器是否支持 web worker
    • 创建web worker文件
    • 创建web worker对象

    # iframe

    概念：会创建包含另外一个文档的行内框架

    优点

    • 加载速度较慢的内容
    • 脚本并行下载
    • 实现跨子域通信

    缺点

    • 阻塞主页面onload事件
    • 无法被一些搜索引擎识别
    • 产生多个页面，不易管理

    # 渐进增强和优雅降级的区别

    两者区别

    优雅降级：

    • 从复杂现状开始，减少供给
    • 针对最高级浏览器来设计网站，有功能缺失的浏览器开发工作留到最后
    • 测试对象为主流浏览器的前一个版本

    渐进增强：打造以内容为主设计网站

    渐进增强

    先对低版本浏览器进行页面重构，再对高级浏览器进行效果交互（往前看）

    优雅降级

    先构建完整功能，再对低版本浏览器兼容（往回看）
