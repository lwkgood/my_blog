    # display:none || visibility: hidden

    共同点

    隐藏元素不可见

    区别

    一、渲染树中

    • 作用：利用CSS 把HTML按照 布局与样式 显示出来
    • display:none 元素会消失，不占据空间
    • visibility:hidden 元素不会消失，占据空间，内容不可见

    二、是否是继承属性

    • display:none 非继承属性 子孙节点随父节点消失，修改子孙节点不显示
    • visibility:hidden 继承属性 子孙节点消失，可设置visibility:visible 显示

    三、修改 display 文档-重排；修改 visibility 元素-重绘

    四、读屏器 display:none 不读取，visibility:hidden 读取

    # CSS 优化和提高性能的方法有哪些

    加载性能

    一、CSS压缩
    二、CSS 单一样式：需要下边距和左边距时，margin-bottom:bottom;margin-left:left，执行效率更高
    三、减少使用@import，改用link

    选择器性能

    一、少用后代选择器，因为CSS选择符是从右到左匹配，会遍历所有子元素
    二、避免使用通配符，计算次数太多
    三、少对标签进行选择，而用class
    四、了解哪些属性可继承，避免重复

    渲染性能

    一、慎用浮动、定位
    二、减少页面重排、重绘
    三、选择器优化嵌套，避免层级过深

    可维护性、健壮性

    一、相同属性的样式抽离，通过 class 在页面使用
    二、样式与内容分离：css 代码定义到外部 css

    # 媒体查询理解

    针对不同的媒体类型定义不同的样式，不改变内容本身，允许内容在特定范围的设备裁剪，让网页对不同型号设备做响应适配

    # CSS工程化理解与实践

    理解

    1.怎么编写CSS代码更好？
    2.怎么打包最优？
    3.后期怎么维护？

    实践

    一、预处理器 Sass || Less

    • 类CSS代码 通过预处理器 转化为 CSS代码

    特性：
    1.嵌套代码
    2.定义css变量
    3.计算函数
    4.循环语句

    二、后处理器 - PostCss

    • 旧CSS代码 通过PostCss 转化为 新CSS代码

    作用：
    1.可做预处理器类似的工作
    2.插件帮助CSS代码适配低版本浏览器
    3.可编译浏览器不支持的高级CSS语法

    三、Webpack loader

    • Webpack 本身不可处理CSS，但在loader辅助下 可处理

    Webpack 使用 css-loader 和 style-loader 操作CSS

    1.css-loader：导入CSS模块，对代码编译处理
    2.style-loader：创建style标签，把CSS内容写入标签
    3.css-loader 执行顺序在 style-loader 前面（先编译后插入）

    # CSS 选择器 || 优先级

    选择器

    1.id选择器 #id 100
    2.类选择器 .classname 10
    3.属性选择器 a[ref='eee'] 10
    4.伪类选择器 li:last-child 10
    5.标签选择器 div 1
    6.伪元素选择器 li::after 1
    7.相邻兄虎选择器 h1+p 0
    8.子选择器 ul>li 0
    9.后代选择器 li a 0
    10.通配符选择器 * 0

    优先级

    !important > 内联样式(1000) > id选择器 > 类、伪类、属性选择器 > 标签、伪元素选择器 > 子代、后代、相邻兄弟、通配符选择器

    # display block、inline、inline-block 区别

    block

    独占一行，自动换行，可设置宽高、外内边距

    inline

    不独占一行，不自动换行，设置宽高、垂直方向的外内边距-无效，可设置水平方向的外内边距

    inline-block

    对象设置为inline对象，但对象内容作为block对象呈现，之后的内联对象会被排列在同一行

    # link || @import 区别

    共同点

    都是外部引用CSS方式

    区别

    1.link 是XHTML标签，可加载CSS、定义RSS等事务；@import 只能加载CSS
    2.link 引用CSS，页面载入时同时加载；@import 页面完全载入后加载
    3.link 无兼容问题；@import 低版本浏览器不支持
    4.link 支持使用JS控制DOM改变样式；@import不支持

    # 盒模型理解

    组成：margin、padding、border、content
    标准盒模型：width、height范围 只包含 content；box-sizing：content-box
    IE盒模型：width、height范围 包含 border、padding、content；box-sizing：border-box

    # li与li之间有看不见的空白间隔

    浏览器会把内联元素间的空白字符（空格、换行、Tab等）渲染成一个空格

    解决方法

    1.为li设置float:left (有些容器不能设置浮动 - 左右切换焦点图)
    2.所有<li>写在同一行

    # px、em、rem 的区别及使用场景

    一、
    px 固定像素，设置了就无法随页面大小而改变 --> 分辨率对页面影响不大时，可用

    二、
    em、rem 相对长度单位，长度不固定，适用于响应式布局

    1.em 相对于父元素设置字体大小（必须知道它父元素大小 - 不便利）
    2.rem 相对于根元素，只需在根元素确定一个参考值

    rem 适配各种移动设备

    四、圣杯布局

    利用浮动和负边距来实现。父级元素设置左右的padding，三列均设置向左浮动，中间一列放在最前面，宽度设置为父级元素的宽度，因此后面两列都被挤到下一行，通过设置margin负值将其移动到上一行，再利用相对定位，定位到两边

    <style>
    .parent {
        padding-left: 100px;
        padding-right: 100px;
        height: 100px;
    }
    .center {
        float: left;
        height: 100px;
        width: 100%;
        background-color: blue;
    }
    .left {
        float: left;
        height: 100px;
        width: 100px;
        background-color: red;
        margin-left: -100%;
        position: relative;
        left: -100px;
    }
    .right {
        float: left;
        height: 100px;
        width: 100px;
        background-color: green;
        margin-left: -100px;
        position: relative;
        left: 100px;
    }
</style>

    # flex布局

    flex 布局是 CSS3 新增的布局方式，将元素的 display 属性值设置为 flex，它成为一个 flex 容器，它的所有子元素都会成为它的项目

    行内元素也可以使用 flex 布局

    .box {
    display: inline-flex;
}
    1
    2
    3

    对于容器，默认有两条轴：一个是水平主轴，一个是与主轴垂直的交叉轴:

    1.flex-direction 指定主轴的方向
    2.justify-content 指定元素在主轴上的排列方式
    3.align-items 指定元素在交叉轴上的排列方式
    4.flex-wrap 规定换行方式

    对于容器中的项目：

    1.flex-grow：如果有剩余空间，项目是否扩大，0为不扩大
    2.flex-shrink：如果剩余空间不够，项目是否缩小，1为缩小
    3.flex-basis：项目本身大小，默认值为auto
    4.order：指定项目的排列顺序

    # flex:1

    flex 属性是 flex-grow、flex-shrink、flex-basis 的简写，默认值为 0 1 auto

    flex:1 --> 不管内容多少，一般都是平分空间，空间大小都一致

    flex-grow: 1;
flex-shrink: 1;
flex-basis: 0%
    1
    2
    3

    flex:auto --> 根据内容的大小来分，不是均分的（除非内容都是一样，才均分）

    flex-grow: 1;
flex-shrink: 1;
flex-basis: auto;
    1
    2
    3

    flex:0 --> 不可扩大，可缩小，表现形式为最小内容宽度,

    flex-grow: 0;
flex-shrink: 1;
flex-basis: 0%;
    1
    2
    3

    flex:none --> 不可扩大，不可缩小，内容本身的宽度是多少就是多少

    # 如何根据设计稿进行移动端适配

    适配不同像素密度

    使用CSS媒体查询，选择不同精度图片，保证图片不失真

    适配不同屏幕大小

    不同屏幕有不同的逻辑像素大小，px固定像素 失真，采用相对单位 rem、em、vw、vh

    # reflow回流 || repaint重绘

    回流：通过构造渲染树，我们将可见DOM节点以及它对应的样式结合起来，可是我们还需要计算它们在设备视口(viewport)内的确切位置和大小，这个计算的阶段就是回流

    重绘：当我们对 DOM 的修改导致了样式的变化、却并未影响其几何属性（比如修改了颜色或背景色）时，浏览器不需重新计算元素的几何属性、直接为该元素绘制新的样式，这个过程叫做重绘

    回流：页面再次构建 render 树。每个页面至少发生一次回流，就是第一次加载页面的时候此外，当页面改变造成文档结构改变，都会回流

    1.添加或删除元素(opacity:0 除外，它不是删除)
    2.改变某个元素的尺寸或位置
    3.浏览器窗口改变（resize事件触发）

    重绘：根据 render 树绘制页面，性能损耗比回流小。每次回流一定重绘

    1.元素的颜色、透明度改变
    2.text-align
