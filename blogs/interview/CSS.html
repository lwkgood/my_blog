    # display:none || visibility: hidden

    共同点

    隐藏元素不可见

    区别

    一、渲染树中

    • 作用：利用CSS 把HTML按照 布局与样式 显示出来
    • display:none 元素会消失，不占据空间
    • visibility:hidden 元素不会消失，占据空间，内容不可见

    二、是否是继承属性

    • display:none 非继承属性 子孙节点随父节点消失，修改子孙节点不显示
    • visibility:hidden 继承属性 子孙节点消失，可设置visibility:visible 显示

    三、修改 display 文档-重排；修改 visibility 元素-重绘

    四、读屏器 display:none 不读取，visibility:hidden 读取

    # 伪类-伪元素 区别 应用

    伪类

    # CSS 优化和提高性能的方法有哪些

    加载性能

    一、CSS压缩
    二、CSS 单一样式：需要下边距和左边距时，margin-bottom:bottom;margin-left:left，执行效率更高
    三、减少使用@import，改用link

    选择器性能

    一、少用后代选择器，因为CSS选择符是从右到左匹配，会遍历所有子元素
    二、避免使用通配符，计算次数太多
    三、少对标签进行选择，而用class
    四、了解哪些属性可继承，避免重复

    渲染性能

    一、慎用浮动、定位
    二、减少页面重排、重绘
    三、选择器优化嵌套，避免层级过深

    可维护性、健壮性

    一、相同属性的样式抽离，通过 class 在页面使用
    二、样式与内容分离：css 代码定义到外部 css

    # display block、inline、inline-block 区别

    block

    独占一行，自动换行，可设置宽高、外内边距

    inline

    不独占一行，不自动换行，设置宽高、垂直方向的外内边距-无效，可设置水平方向的外内边距

    inline-block

    对象设置为inline对象，但对象内容作为block对象呈现，之后的内联对象会被排列在同一行

    # link || @import 区别

    共同点

    都是外部引用CSS方式

    区别

    1.link 是XHTML标签，可加载CSS、定义RSS等事务；@import 只能加载CSS
    2.link 引用CSS，页面载入时同时加载；@import 页面完全载入后加载
    3.link 无兼容问题；@import 低版本浏览器不支持
    4.link 支持使用JS控制DOM改变样式；@import不支持

    # px、em、rem 的区别及使用场景

    一、
    px 固定像素，设置了就无法随页面大小而改变 --> 分辨率对页面影响不大时，可用

    二、
    em、rem 相对长度单位，长度不固定，适用于响应式布局

    1.em 相对于父元素设置字体大小（必须知道它父元素大小 - 不便利）
    2.rem 相对于根元素，只需在根元素确定一个参考值

    rem 适配各种移动设备

    # flex布局

    flex 布局是 CSS3 新增的布局方式，将元素的 display 属性值设置为 flex，它成为一个 flex 容器，它的所有子元素都会成为它的项目

    行内元素也可以使用 flex 布局

    .box {
    display: inline-flex;
}
    1
    2
    3

    对于容器，默认有两条轴：一个是水平主轴，一个是与主轴垂直的交叉轴:

    1.flex-direction 指定主轴的方向
    2.justify-content 指定元素在主轴上的排列方式
    3.align-items 指定元素在交叉轴上的排列方式
    4.flex-wrap 规定换行方式

    对于容器中的项目：

    1.flex-grow：如果有剩余空间，项目是否扩大，0为不扩大
    2.flex-shrink：如果剩余空间不够，项目是否缩小，1为缩小
    3.flex-basis：项目本身大小，默认值为auto
    4.order：指定项目的排列顺序

    # flex:1

    flex 属性是 flex-grow、flex-shrink、flex-basis 的简写，默认值为 0 1 auto

    flex:1 --> 不管内容多少，一般都是平分空间，空间大小都一致

    flex-grow: 1;
flex-shrink: 1;
flex-basis: 0%
    1
    2
    3

    flex:auto --> 根据内容的大小来分，不是均分的（除非内容都是一样，才均分）

    flex-grow: 1;
flex-shrink: 1;
flex-basis: auto;
    1
    2
    3

    flex:0 --> 不可扩大，可缩小，表现形式为最小内容宽度,

    flex-grow: 0;
flex-shrink: 1;
flex-basis: 0%;
    1
    2
    3

    flex:none --> 不可扩大，不可缩小，内容本身的宽度是多少就是多少

    # 如何根据设计稿进行移动端适配

    适配不同像素密度

    使用CSS媒体查询，选择不同精度图片，保证图片不失真

    # reflow回流 || repaint重绘

    回流：通过构造渲染树，我们将可见DOM节点以及它对应的样式结合起来，可是我们还需要计算它们在设备视口(viewport)内的确切位置和大小，这个计算的阶段就是回流

    重绘：当我们对 DOM 的修改导致了样式的变化、却并未影响其几何属性（比如修改了颜色或背景色）时，浏览器不需重新计算元素的几何属性、直接为该元素绘制新的样式，这个过程叫做重绘

    回流：页面再次构建 render 树。每个页面至少发生一次回流，就是第一次加载页面的时候此外，当页面改变造成文档结构改变，都会回流

    1.添加或删除元素(opacity:0 除外，它不是删除)
    2.改变某个元素的尺寸或位置
    3.浏览器窗口改变（resize事件触发）

    重绘：根据 render 树绘制页面，性能损耗比回流小。每次回流一定重绘

    1.元素的颜色、透明度改变
    2.text-align

    # position

    一、relative

    生成相对定位的元素，相对于其原来的位置进行定位，元素的位置通过 left、top、right、bottom 规定

    二、absolute

    生成绝对定位的元素，相对于 static 定位以外的一个父元素进行定位，元素的位置通过 left、top、right、bottom 规定

    三、fixed

    生成绝对定位的元素，指定元素相对于屏幕视⼝（viewport）的位置来指定元素位置，元素的位置在屏幕滚动时不会改变

    元素的定位是相对于 window （或者 iframe）边界的

    四、static

    默认值，没有定位，元素出现在正常的文档流中，会忽略 top, bottom, left, right 或者 z-index 声明

    # margin重叠

    两个块级元素的上外边距和下外边距可能会合并为一个外边距，其大小会取其中外边距值大的那个，这种行为就是外边距折叠 - 垂直方向

    浮动元素、绝对定位 --> 不会外边距折叠

    解决方法

    一、兄弟之间重叠

    1.底部元素变为行内元素：display: inline-block

    # BFC

    块级格式化上下文，是一个独立的布局环境，类似于一个容器，在容器按照规则摆放物品，并且不影响其他环境中的物品，如果一个元素触发BFC，则BFC中的元素布局不受外部影响

    创建BFC

    1.根元素：body
    2.元素设置浮动：float 除 none 以外的值
    3.元素设置绝对定位：position (absolute、fixed)
    4.display为：inline-block、table-cell、flex
    5.overflow为：hidden、auto、scroll

    1.解决 margin 的重叠问题：BFC 是一个独立区域，内部元素和外部元素互不影响，将两个元素变为两个BFC即可
    2.解决高度塌陷的问题：在对子元素设置浮动后，父元素会发生高度塌陷，即父元素高度变为 0（解决方法：只需把父元素变成BFC，即给父元素设置overflow:hidden）

    # clear清除浮动原理

    clear: none | left | right | both

    1.clear属性：元素盒子的边不能和前面的浮动元素相邻，对元素设置 clear 是为了避免浮动元素对该元素的影响，而不是清除浮动
    2.clear属性对“后面的”浮动元素是不闻不问的，float要么left要么right，不可能同时存在，直接clear:both
    3.一般使用伪元素的方式清除浮动

    .clear::after {
    content: "";
    display: block;
    clear: both;
}

    # 浮动 || 清除方式

    浮动

    1.容器不设高度且子元素浮动时，容器高度不能被内容撑开， 内容会溢出到容器外面而影响布局
    2.脱离文档流，不占据空间 - 引起高度塌陷现象
    3.浮动元素碰到包含它的边框或者其他浮动元素的边框停留

    清除方式

    1.给父级 div 定义height属性
    2.最后一个浮动元素之后添加一个空的 div 标签，并添加clear:both样式
    3.包含浮动元素的父级标签添加overflow:hidden或者overflow:auto
    4.使用:after伪元素，由于 IE6-7 不支持:after，使用 *zoom:1

    .clearfix:after {
    content: "";
    display: table;
    height: 0;
    clear: both;
}
.clearfix {
    *zoom: 1;
}

    # 设置小于12px的字体

    1.谷歌下 css 设置字体大小为 12px 及以下，显示都是默认 12px
    2.使用图片；如果是内容固定不变情况下，使用将小于 12px 文字内容切出做图片
    3.使用 css3 的 transform 缩放属性-webkit-transform:scale(0.5)；它收缩的是整个元素的大小，若是内联元素，则要转换成块元素

    <style>
    div {
        background-color: red;
        margin-bottom: 5px;
    }
    .one {
        font-size: 12px;
    }
    .two {
        font-size: 8px;
        transform: scale(0.6666); /*要设置8px，那就是8/12=0.66666*/
        /*
            默认缩放中心点是在盒子的正中心，所以如果我们需要文本左对齐，就需要改变中心点，也就是transform-origin：0 0；
            该值有两个参数值，第一个是水平方位值，第二个是垂直方位值
        */
        transform-origin: 0 0;
        /*
            文本进行缩放后，并不会改变其原来盒子的大小，只是视觉上改变了大小，也就是说如果文本有换行的时候，它进行缩放后仍然是折行显示，这显然不符合我们要求，
            所以我们还需要强制文本在一行显示
        */
        white-space: nowrap;
    }
</style>
<body>
    <div class="one">哈哈哈 12px 字体大小 Hello</div>
    <div class="two">哈哈哈 8px 字体大小 Hello</div>
</body>
